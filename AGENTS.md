# AGENTS.md — BEStie project landing site

Context for continuing work in a new session.

## What this repo is

GitHub Pages **organization site** for `bestie-org`, live at `https://bestie-org.github.io/`.
Its primary business goal is **search-engine visibility for the BEStie project**
(https://github.com/bestie-org/BEStie) — Google/Bing verification is not possible on a bare
GitHub repo, hence this site. Audience: **non-technical ebike riders**.

Content is adapted from the BEStie README (https://raw.githubusercontent.com/bestie-org/BEStie/refs/heads/master/README.md).

## Tech stack

- **Jekyll, no theme** — built natively by GitHub Pages on push to `master`. No CI.
- Plugins: `jekyll-seo-tag`, `jekyll-sitemap` (both GH Pages whitelisted).
- Single hand-written stylesheet, no frameworks. Only JS: `assets/js/stars.js`.
- Local build: `jekyll build` / `jekyll serve` (user-installed gems; `Gemfile` is for local
  convenience only — Pages ignores it).

## Layout

```
_config.yml          # site meta + search-engine verification tokens (webmaster_verifications:)
_layouts/default.html
_includes/github-button.html   # "View on GitHub" + live star count chip
index.md             # landing: pitch, Why BEStie, how-it-works, star CTA, SoftwareApplication JSON-LD
hardware.md          # board photo cards (Feather / Nordic DK / XIAO), bike + fitness gear compat
installation.md      # 6-step guide, videos hotlinked from github.com/user-attachments
faq.md               # <details> accordions + FAQPage JSON-LD
assets/css/style.css
assets/js/stars.js   # GitHub API stargazers_count, 1h localStorage cache, silent fallback
assets/img/          # self-hosted hero, og-image, board photos
robots.txt           # references sitemap.xml (generated by jekyll-sitemap)
favicon.svg
```

## Key decisions (don't undo without asking)

- **Verification tokens** live ONLY in `_config.yml` → `webmaster_verifications` (google/bing).
  Empty values are skipped by jekyll-seo-tag. Both tokens are currently set.
- **Meta descriptions are capped at ≤150 chars** (Bing Webmaster Tools requirement) — keep every
  page `description:` and the site fallback within that limit.
- **"View on GitHub" button, NOT a star button**: GitHub has no star-via-link mechanism; an
  earlier "Star on GitHub" label was replaced as misleading. Star count chip stays (stars.js).
- **Brand names (Garmin, Wahoo) intentionally sprinkled** for SEO. Guardrail: only Garmin devices
  are author-tested; Wahoo is cited via the BLE FTMS smart-trainer standard, never as "tested".
- **Board photos are vendor product photos** (Nordic, Adafruit, Seeed), self-hosted with
  attribution in card captions + footer disclaimer. Formally copyrighted, used editorially —
  owner's call, replace if a vendor objects.
- **Videos/images hotlinked** from `github.com/user-attachments/...` (same assets the README uses).
  HEAD requests 403 on these; GET works (verified 206 video/mp4).
- Footer carries a non-affiliation disclaimer (Bosch, Garmin, Wahoo, Nordic, Adafruit, Seeed).

## Deploy

Push to `master` → GitHub Pages rebuilds (~1 min). The local user pushes themselves;
agent push from the harness failed previously (SSH env restrictions) — do not retry blindly.

## Gotchas

- Search Console "Couldn't fetch" on a fresh sitemap is normal; it means "not fetched yet",
  not an error. Sitemap verified valid: 200, `application/xml`, Googlebot-accessible.
- README FAQ numbering skips #6 (source quirk); the site renumbers 1–7.
- The firmware uploader at `bestie-org.github.io/adafruit-uploader/` is a **separate project
  repo's Pages site** — do not create a conflicting `adafruit-uploader/` path here.
- `Gemfile.lock` and `_site/` are gitignored.
